## Tools

List every tool your agent will use. For each tool, fill in all four fields.
You must have at least 3 tools. The three required tools are listed — add any additional tools below them.

### Tool 1: search_listings

**What it does:**
- Searches local listings database to find secondhand apparel items that match the user's keywords, size, and maximum budget.
- Ranks filteered results based on keyword overlap with item descriptions.

**Input parameters:**
- `description` (str): keywords or style tags that describe what the user is looking for
- `size` (str): sizing of item
- `max_price` (float): maximum budget that user is willing to spend

**What it returns:**
- A list of dictionaries representing matching listings sorted by relevance (highest keyword overlap score first).
- Each dictionary contains the fields: id, title, description, category, style_tags, size, condition, price, colors, brand, platform.

**What happens if it fails or returns nothing:**
- If no item match the criteria, an empty list [] is returned instead of a raised exception.
- The agent loop should understand the empty list as a signal to stop the styling process and return a grounding message.

---

### Tool 2: suggest_outfit

**What it does:**
- Pairs the selected secondhand item with items from the user's current wardrobe to create 1–2 cohesive outfit suggestions using an LLM.
- If the user's wardrobe is empty, it uses the LLM to supply general styling advice and aesthetic vibes for the new item instead.

**Input parameters:**
- `new_item` (dict): dictionary with the new thrifted item that the user is considering
- `wardrobe` (dict): dictionary containing items that user currently owns

**What it returns:**
- A string that specifies a creative outfit combination or styling advice.

**What happens if it fails or returns nothing:**
- If the wardrobe is empty, the tool should pivot to providing general styling advice. Otherwise, it provides aesthetic advice specific to the item that may be thrifted.

---

### Tool 3: create_fit_card

**What it does:**
- Takes a generated outfit suggestion and the details of the thrifted item to compile a trendy, short, and highly shareable social media caption.

**Input parameters:**
- `outfit` (str): description generated by `suggest_outfit` that provides user with caption
- `new_item` (dict): dictionary with the new thrifted item that the user is considering

**What it returns:**
- A casual, authentic 2–4 sentence string optimized to look like an Instagram or TikTok OOTD post caption that incorporates the item name, price, and shopping platform.

**What happens if it fails or returns nothing:**
- If the outfit string is missing, empty, or whitespace-only, the tool guards against crashing by returning a clean, descriptive error message string instead of raising an exception.

## Planning Loop

**How does your agent decide which tool to call next?**
The loop begins by checking the user's incoming query. If it's empty, the system sets an error message and terminates immediately. If the user inputs a valid string, the system extracts parameters like `size` and `max_price` before invoking `search_listings(description, size, max_price)`. If an empty list is returned, an error message is logged to `session["error"]` and execution halts. If the list has matching items, the agent stores them in `session["results"]` and extracts the top item as `session["selected_item"]`. Subsequently, the agent calls `suggest_outfit(new_item, wardrobe)`, which maps items from the user's closet to pair with the thrifted clothing or provides general styling advice if the wardrobe is empty. This output is saved in session["outfit_suggestion"] and passed to `create_fit_card(outfit, new_item)`. If the outfit input is missing or empty, an error string is safely assigned. Once a proper caption is generated and saved to `session["fit_card"]`, the agent terminates and presents the data to the UI.

---

## State Management

**How does information from one tool get passed to the next?**
The agent manages data with a mutable dictionary `session`. It tracks the query, wardrobe, results, selected_item, outfit_suggestion, fit_card, and error. When the user puts their initial query, it extracts necessary details and then updates session's parameters respectively.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Sets `session["error"]` to a helpful message suggesting the user loosen their filters, then exits the loop early to block downstream tool execution. |
| suggest_outfit | Wardrobe is empty | Pivots the LLM prompt to generate general aesthetic guidelines and styling inspiration for the item rather than crashing.|
| create_fit_card | Outfit input is missing or incomplete | Catches the error condition and returns a descriptive error caption string instead of raising an unhandled exception. |

### Stretch Feature: Retry Logic with Fallback
If `search_listings` returns an empty list `[]`, the agent will not immediately fail. Instead, it enters a fallback state:
1. It attempts a second search completely omitting the `size` filter (widening the pool).
2. If that still returns nothing, it attempts a third search by removing both `size` and increasing the `max_price` budget constraint by 50%.
3. The agent logs these adjustments in `session["error"]` as a warning notification to the user so they know they are viewing fallback results.
